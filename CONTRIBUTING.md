# Contributing to Spar Nord Bank ETL Project

Thank you for your interest in contributing to this ETL project!

## How to Contribute

### Reporting Issues
- Describe issues clearly
- Include error logs if available
- Specify environment details

### Submitting Changes

1. Fork the repository
2. Create a feature branch: `git checkout -b feature/your-feature`
3. Install dependencies: `pip install -r requirements.txt`
4. Test thoroughly with sample data
5. Make your improvements
6. Push and submit a Pull Request

## Code Guidelines

- Follow PEP 8 conventions
- Document ETL processes
- Include data validation logic
- Optimize for performance

## Areas for Contribution

- Improving ETL efficiency
- Better error handling
- Enhanced logging
- Performance optimization
- Documentation improvements
- New data pipeline steps

## Pull Request Process

1. Test with sample datasets
2. Include performance metrics
3. Document changes
4. Update README if needed

Thank you for contributing! 🙏
